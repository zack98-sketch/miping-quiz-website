# 需求规格文档 - 密评题库答题网站

## 1. 概述

### 1.1 项目描述
密评题库答题网站是一个面向编程学习者、学生和技术人员的在线编程练习平台。系统基于包含5075道题的密评题库（含答案解析），提供多种答题模式、错题本管理、收藏功能、学习分析、在线考试、AI智能提示等核心功能，帮助用户系统化地学习和掌握密码学相关知识。系统支持移动端适配、Docker容器化部署，项目代码托管于GitHub并建立完整的版本管理与更新记录机制。

### 1.2 术语定义

| 术语 | 定义 |
|------|------|
| 密评题库 | 包含5075道密码学相关题目的题库，题型包括单选题、多选题、判断题 |
| 答题会话 | 用户一次完整的答题过程，包含选定的题目集合、用户作答记录及评判结果 |
| 错题本 | 系统自动收集的用户答错题目集合，支持查看、编辑、删除和重做 |
| 收藏夹 | 用户主动标记的题目集合，用于重点复习 |
| 知识点 | 题目的分类标签，如"数据结构"、"算法"、"Python异常处理"等 |
| 题型 | 题目的作答形式，包括单选题、多选题、判断题 |
| 难度 | 题目的难易程度等级 |
| 答题模式 | 系统提供的出题策略，包括单题模式、混合模式、知识点模式、难度模式 |
| 答题记录 | 用户每次答题的完整历史记录，包括答题时间、答案、正确与否、用时等 |
| 答案解析 | 题目附带的答案解释说明，用于错题反馈和学习参考 |
| 在线考试 | 管理员创建的限时答题活动，包含考试名称、时间限制、题目配置和参与人员 |
| AI智能提示 | 系统基于AI模型为用户提供的答题辅助提示，包括知识点提示和解题思路引导 |
| 题目纠错 | 用户对题目内容、答案或解析中的错误提交修正建议的功能 |
| 题目备注 | 用户对特定题目添加个人笔记或理解记录的功能 |
| 移动端适配 | 系统界面在移动设备（手机、平板）上的响应式适配与优化 |

### 1.3 系统范围

- **包含**:
  - 题库数据导入与管理（管理员功能）
  - 多模式答题系统（单题模式、混合模式、知识点模式、难度模式）
  - 实时判题与错题反馈
  - 错题本管理（查看、编辑、删除、重做）
  - 收藏夹管理（收藏、查看、练习）
  - 答题统计与学习分析
  - 答题记录导出（PDF/Word）
  - 题目纠错功能（用户提交纠错、管理员审核处理）
  - 题目备注功能（用户添加/编辑/查看个人备注）
  - 在线考试功能（管理员创建考试、用户参加考试、考试结果统计）
  - AI智能提示功能（答题辅助提示、知识点提示、解题思路引导）
  - 移动端适配（响应式布局、触控优化、移动端专属交互）
  - 用户认证与授权
  - Docker容器化部署
  - HTTPS安全访问
  - GitHub项目管理与代码更新记录

- **不包含**:
  - 多用户实时对战/竞赛
  - 题目在线编辑器（普通用户不可直接编辑题库）
  - 社交功能（评论、分享、排行榜）

## 2. 需求区域

### 2.1 题库管理

#### 2.1.1 功能需求

- **REQ-TM-001**: 系统应支持从Excel文件（.xlsx）导入题库数据，包括题目内容、题型、选项、正确答案、答案解析、知识点标签和难度等级。
  - **验收标准**: 导入5075道题后，系统数据库中题目数量与Excel文件一致，所有字段完整无缺失。

- **REQ-TM-002**: 当管理员上传新的题库文件时，系统应解析文件内容并验证数据完整性，对格式错误或字段缺失的记录生成错误报告。
  - **验收标准**: 上传含格式错误的文件时，系统返回具体错误信息（行号、字段名、错误原因），正确记录正常导入。

- **REQ-TM-003**: 系统应支持题库数据的初始化导入，管理员可通过命令行工具或管理界面执行一次性批量导入。
  - **验收标准**: 执行初始化导入命令后，题库数据完整写入数据库，可通过API查询验证。

- **REQ-TM-004**: 系统应确保普通用户无法直接编辑、修改或删除题库原始内容。
  - **验收标准**: 普通用户角色的题库编辑API请求返回403禁止访问，管理界面不显示题库编辑入口。

- **REQ-TM-005**: 系统应支持按题型、知识点、难度对题库进行分类索引，以支持高效筛选出题。
  - **验收标准**: 按任意条件组合筛选，返回结果在500ms内响应，结果准确无遗漏。

- **REQ-TM-006**: 系统应提供题库统计信息接口，返回总题数、各题型数量、各知识点数量和各难度数量。
  - **验收标准**: 统计数据与实际题库数据一致，接口响应时间不超过200ms。

#### 2.1.2 非功能需求

- **NFR-TM-001**: 题库导入操作应在60秒内完成5075道题的批量写入。
- **NFR-TM-002**: 题库查询接口响应时间应不超过200ms（95分位）。

### 2.2 答题系统

#### 2.2.1 功能需求

- **REQ-QZ-001**: 当用户选择开始答题时，系统应根据用户选择的答题模式和参数生成答题会话，并返回题目列表。
  - **验收标准**: 用户选择模式并设置参数后，系统返回符合筛选条件的题目集合，题目数量与用户设置一致。

- **REQ-QZ-002**: 系统应支持随机出题模式，用户可设置出题数量（如10题、20题、50题），系统从题库中随机抽取指定数量的题目。
  - **验收标准**: 设置出10题时返回10道不重复的题目；连续两次随机出题，题目序列不完全相同。

- **REQ-QZ-003**: 系统应支持按题型筛选出题，用户可选择只出单选题、只出多选题、只出判断题或多种题型组合。
  - **验收标准**: 选择"仅单选题"时，返回的题目全部为单选题；选择组合题型时，各题型均有题目。

- **REQ-QZ-004**: 系统应支持按知识点筛选出题，用户可选择一个或多个知识点标签，系统仅从对应知识点范围的题目中出题。
  - **验收标准**: 选择"数据结构"知识点时，所有返回题目的知识点标签均包含"数据结构"。

- **REQ-QZ-005**: 系统应支持按难度筛选出题，用户可选择难度等级，系统仅从对应难度的题目中出题。
  - **验收标准**: 选择"困难"难度时，所有返回题目的难度等级均为"困难"。

- **REQ-QZ-006**: 系统应在答题界面清晰展示当前题目的完整信息，包括题号、题目文本、选项（单选/多选）或判断选项（正确/错误）。
  - **验收标准**: 每道题的展示包含所有必要信息，选项布局清晰可辨，单选题为单选控件，多选题为多选控件，判断题为判断控件。

- **REQ-QZ-007**: 当用户提交单题答案时，系统应立即评判答案正确与否，并显示评判结果。
  - **验收标准**: 提交答案后1秒内显示"正确"或"错误"标识。

- **REQ-QZ-008**: 当用户答错题目时，系统应展示该题的正确答案和答案解析。
  - **验收标准**: 答错后界面显示正确答案高亮标注，并展示完整的答案解析文本。

- **REQ-QZ-009**: 当用户完成答题会话中所有题目时，系统应生成本次答题总结，包括总题数、正确数、错误数、正确率、总用时和错题详情列表。
  - **验收标准**: 答题结束后展示完整总结页面，正确率计算准确，错题详情包含题目、用户答案、正确答案和解析。

- **REQ-QZ-010**: 系统应支持单题模式答题，即逐题展示，用户答完当前题后才进入下一题。
  - **验收标准**: 答题过程中仅显示当前一题，提交答案后才可查看下一题，不可跳题。

- **REQ-QZ-011**: 系统应支持混合模式答题，即系统自动从全部题库中随机出题，不限定题型、知识点或难度。
  - **验收标准**: 选择混合模式后，题目从全题库随机抽取，题型分布与题库整体分布一致。

- **REQ-QZ-012**: 系统应支持知识点模式答题，即用户先选择知识点，系统从该知识点范围的题目中出题。
  - **验收标准**: 选择知识点模式后，系统展示知识点选择界面，确认后从选定范围出题。

- **REQ-QZ-013**: 系统应支持难度模式答题，即用户先选择难度等级，系统从该难度范围的题目中出题。
  - **验收标准**: 选择难度模式后，系统展示难度选择界面，确认后从选定范围出题。

- **REQ-QZ-014**: While 用户处于答题过程中，系统应记录每道题的作答时间戳和累计用时。
  - **验收标准**: 答题总结中的用时数据与实际答题时间一致，精确到秒。

- **REQ-QZ-015**: 当用户在答题过程中退出或中断时，系统应保存当前答题进度，用户重新进入时可选择继续答题或重新开始。
  - **验收标准**: 中途退出后重新进入，系统提示"检测到未完成的答题，是否继续"，选择继续则恢复到中断时的题目。

#### 2.2.2 非功能需求

- **NFR-QZ-001**: 答题页面加载时间应不超过1秒。
- **NFR-QZ-002**: 系统应支持至少100个用户同时在线答题而不出现性能下降。
- **NFR-QZ-003**: 答题判题响应时间应不超过500ms。

### 2.3 错题本与学习统计

#### 2.3.1 功能需求

- **REQ-ER-001**: 当用户答错一道题时，系统应自动将该题加入用户的错题本，记录答错时间、用户答案、正确答案和答案解析。
  - **验收标准**: 答错后立即在错题本中可查到该题，记录信息完整准确。

- **REQ-ER-002**: 系统应支持用户查看错题本，以列表形式展示所有错题，支持按知识点、题型、答错时间筛选和排序。
  - **验收标准**: 错题本列表正确展示所有历史错题，筛选和排序结果准确。

- **REQ-ER-003**: 系统应支持用户编辑错题本中的记录，包括添加个人笔记和标注掌握状态。
  - **验收标准**: 编辑后信息正确保存并持久化，再次查看时显示更新后的内容。

- **REQ-ER-004**: 系统应支持用户从错题本中删除记录。
  - **验收标准**: 删除后该记录从错题本中移除，不再出现在列表中。

- **REQ-ER-005**: 系统应支持错题重做功能，用户可选择错题本中的部分或全部错题进行专项练习。
  - **验收标准**: 选择错题重做后，系统生成仅包含选定错题的答题会话，答题流程与正常答题一致。

- **REQ-ER-006**: 系统应支持随机错题练习功能，从错题本中随机抽取指定数量的错题进行练习。
  - **验收标准**: 设置抽取数量后，系统从错题本中随机选取对应数量的题目开始答题。

- **REQ-ER-007**: 系统应根据用户答题记录统计各知识点的答错数量和答对数量，计算各知识点的掌握率。
  - **验收标准**: 统计数据与实际答题记录一致，掌握率 = 答对数 / (答对数 + 答错数) × 100%。

- **REQ-ER-008**: 系统应提供答题统计图表，包括正确率趋势图和知识点掌握情况图。
  - **验收标准**: 图表数据与统计计算结果一致，图表可交互查看具体数值。

- **REQ-ER-009**: 当用户在错题重做中答对某题时，系统应提示用户是否将该题从错题本中移除。
  - **验收标准**: 答对后弹出确认提示，用户确认后该题从错题本移除。

#### 2.3.2 非功能需求

- **NFR-ER-001**: 错题本查询响应时间应不超过300ms。
- **NFR-ER-002**: 统计图表渲染时间应不超过2秒。

### 2.4 收藏功能

#### 2.4.1 功能需求

- **REQ-FV-001**: 当用户在答题或查看题目时点击收藏按钮，系统应将该题加入用户的收藏夹。
  - **验收标准**: 收藏后题目出现在收藏夹列表中，收藏按钮状态变为"已收藏"。

- **REQ-FV-002**: 系统应支持用户查看收藏夹，以列表形式展示所有收藏的题目，支持按知识点、题型、收藏时间筛选和排序。
  - **验收标准**: 收藏夹列表正确展示所有收藏题目，筛选和排序结果准确。

- **REQ-FV-003**: 系统应支持用户取消收藏，将题目从收藏夹中移除。
  - **验收标准**: 取消收藏后该题从收藏夹中移除，收藏按钮状态恢复为"未收藏"。

- **REQ-FV-004**: 系统应支持用户从收藏夹中选择题目进行练习，可选择部分或全部收藏题目。
  - **验收标准**: 选择收藏题目练习后，系统生成包含选定题目的答题会话。

- **REQ-FV-005**: 系统应支持用户查看收藏题目的答案解析。
  - **验收标准**: 在收藏夹中点击题目可查看完整题目信息和答案解析。

#### 2.4.2 非功能需求

- **NFR-FV-001**: 收藏/取消收藏操作响应时间应不超过200ms。
- **NFR-FV-002**: 收藏夹查询响应时间应不超过300ms。

### 2.5 学习分析

#### 2.5.1 功能需求

- **REQ-LA-001**: 系统应提供知识点掌握度分析，对每个知识点计算用户的掌握率并以百分比形式展示。
  - **验收标准**: 展示"你对XX知识点的掌握率为XX%"，数据与答题记录统计一致。

- **REQ-LA-002**: 系统应根据用户错题记录自动生成练习建议，推荐与错题知识点相关的未答题目。
  - **验收标准**: 推荐题目与用户错题的知识点相关，且不包含用户已答对的题目（可选包含）。

- **REQ-LA-003**: 系统应将所有答题记录历史归档，用户可按日期范围查看历史答题记录。
  - **验收标准**: 历史记录按时间倒序展示，支持按日期范围筛选，数据完整无遗漏。

- **REQ-LA-004**: 系统应支持将答题记录导出为PDF格式，包含题目、用户答案、正确答案和解析。
  - **验收标准**: 导出的PDF文件可正常打开，内容完整、格式清晰、支持中文显示。

- **REQ-LA-005**: 系统应支持将答题记录导出为Word格式，包含题目、用户答案、正确答案和解析。
  - **验收标准**: 导出的Word文件可正常打开，内容完整、格式清晰、支持中文显示。

- **REQ-LA-006**: 系统应提供学习进度追踪功能，展示用户累计答题数、累计正确率、学习天数等指标。
  - **验收标准**: 进度数据与历史记录一致，展示清晰直观。

- **REQ-LA-007**: 系统应提供知识点薄弱项分析，按掌握率从低到高排列知识点，标识需要重点复习的知识点。
  - **验收标准**: 掌握率低于60%的知识点标记为"薄弱"，列表按掌握率升序排列。

#### 2.5.2 非功能需求

- **NFR-LA-001**: 学习分析页面加载时间应不超过2秒。
- **NFR-LA-002**: 导出文件生成时间应不超过10秒（单次答题记录）。

### 2.6 用户管理

#### 2.6.1 功能需求

- **REQ-UM-001**: 系统应支持用户注册，用户通过用户名和密码创建账户。
  - **验收标准**: 注册成功后可使用用户名和密码登录，用户名唯一不可重复。

- **REQ-UM-002**: 系统应支持用户登录，验证用户名和密码后发放认证令牌。
  - **验收标准**: 登录成功后跳转至首页，后续请求携带认证令牌。

- **REQ-UM-003**: 系统应支持用户修改密码。
  - **验收标准**: 输入旧密码验证通过后可设置新密码，新密码立即生效。

- **REQ-UM-004**: 系统应区分管理员用户和普通用户角色，管理员拥有题库管理和系统管理权限，普通用户拥有答题和学习功能权限。
  - **验收标准**: 管理员可访问题库管理和系统管理功能，普通用户无法访问；普通用户可正常使用答题功能。

- **REQ-UM-005**: 当用户未登录时访问需认证的页面，系统应重定向至登录页面。
  - **验收标准**: 未登录状态下访问答题页面，自动跳转至登录页。

- **REQ-UM-006**: 系统应支持用户登出，清除认证状态。
  - **验收标准**: 登出后无法访问需认证的页面，需重新登录。

#### 2.6.2 非功能需求

- **NFR-UM-001**: 用户密码应使用加盐哈希存储，禁止明文存储。
- **NFR-UM-002**: 认证令牌有效期应不超过24小时，过期后需重新登录。

### 2.7 题目纠错

#### 2.7.1 功能需求

- **REQ-CR-001**: 当用户发现题目内容、选项、答案或解析存在错误时，系统应支持用户提交纠错报告，包括纠错类型（题目内容错误、选项错误、答案错误、解析错误、其他）、纠错描述和修正建议。
  - **验收标准**: 提交纠错后系统保存纠错记录，用户可在"我的纠错"中查看提交状态。

- **REQ-CR-002**: 系统应支持管理员查看所有待审核的纠错报告列表，按提交时间排序，支持按纠错类型和题目ID筛选。
  - **验收标准**: 管理员可查看完整纠错列表，筛选结果准确。

- **REQ-CR-003**: 当管理员审核纠错报告时，系统应支持通过或驳回操作，通过时需填写修正内容并更新题库，驳回时需填写驳回原因。
  - **验收标准**: 通过后题库对应题目自动更新，驳回后用户收到驳回通知和原因。

- **REQ-CR-004**: 系统应在题目详情页展示该题的纠错历史记录（已通过的纠错），供所有用户参考。
  - **验收标准**: 题目详情页显示纠错修正历史，包含修正时间和修正内容。

- **REQ-CR-005**: 系统应限制同一用户对同一题目在24小时内只能提交一次纠错报告，防止重复提交。
  - **验收标准**: 24小时内重复提交时提示"您已提交过纠错，请等待审核"。

#### 2.7.2 非功能需求

- **NFR-CR-001**: 纠错提交操作响应时间应不超过500ms。
- **NFR-CR-002**: 管理员审核操作响应时间应不超过1秒。

### 2.8 题目备注

#### 2.8.1 功能需求

- **REQ-NB-001**: 系统应支持用户对任意题目添加个人备注，备注内容支持富文本格式（包括文字、重点标记）。
  - **验收标准**: 添加备注后保存成功，在题目详情页可查看备注内容。

- **REQ-NB-002**: 系统应支持用户编辑和删除自己添加的题目备注。
  - **验收标准**: 编辑后备注内容更新，删除后备注移除，操作即时生效。

- **REQ-NB-003**: 系统应在答题界面和错题本中展示用户对该题的备注摘要，点击可展开查看完整备注。
  - **验收标准**: 答题和错题查看时，若该题有备注则显示备注图标和摘要，点击展开完整内容。

- **REQ-NB-004**: 系统应支持用户查看所有题目备注的汇总列表，支持按知识点、题型和添加时间筛选排序。
  - **验收标准**: 备注列表正确展示所有个人备注，筛选和排序结果准确。

- **REQ-NB-005**: 题目备注为用户私有数据，其他用户不可见。
  - **验收标准**: 用户A的备注对用户B不可见，仅备注所有者可查看和编辑。

#### 2.8.2 非功能需求

- **NFR-NB-001**: 备注保存操作响应时间应不超过300ms。
- **NFR-NB-002**: 单条备注内容长度上限为2000字符。

### 2.9 在线考试

#### 2.9.1 功能需求

- **REQ-EX-001**: 当管理员创建在线考试时，系统应支持设置考试名称、考试说明、开始时间、结束时间、时间限制（分钟）和题目配置（题型、数量、知识点范围）。
  - **验收标准**: 创建考试后，考试信息完整保存，管理员可在考试列表中查看。

- **REQ-EX-002**: 系统应根据管理员设置的题目配置，从题库中自动抽取对应数量和条件的题目组成考试试卷。
  - **验收标准**: 试卷题目数量、题型、知识点范围与配置一致，题目随机不重复。

- **REQ-EX-003**: 当用户在考试时间范围内进入考试时，系统应展示考试说明并开始计时，用户答题过程中实时显示剩余时间。
  - **验收标准**: 进入考试后倒计时开始，剩余时间实时更新，时间到自动提交。

- **REQ-EX-004**: 当考试时间结束或用户主动交卷时，系统应自动评判所有题目并生成考试成绩，包括总分、各题型得分、正确率和用时。
  - **验收标准**: 交卷后立即显示考试成绩和详细评判结果。

- **REQ-EX-005**: 系统应支持管理员查看考试结果统计，包括参与人数、平均分、最高分、最低分和分数分布。
  - **验收标准**: 考试结束后管理员可查看完整统计数据和每位参与者的成绩详情。

- **REQ-EX-006**: 系统应支持用户查看自己参加过的考试历史记录和成绩。
  - **验收标准**: 用户可查看历次考试的名称、时间、成绩和答题详情。

- **REQ-EX-007**: 当用户未在考试时间范围内尝试进入考试时，系统应拒绝并提示考试未开始或已结束。
  - **验收标准**: 非考试时间段进入，显示"考试未开始"或"考试已结束"提示。

- **REQ-EX-008**: 系统应确保每位用户在同一考试中只能提交一次答卷。
  - **验收标准**: 已提交答卷的用户再次进入考试，显示已提交的成绩，不可重复答题。

#### 2.9.2 非功能需求

- **NFR-EX-001**: 考试创建操作响应时间应不超过1秒。
- **NFR-EX-002**: 考试交卷评判响应时间应不超过3秒。
- **NFR-EX-003**: 系统应支持至少50人同时参加同一考试。

### 2.10 AI智能提示

#### 2.10.1 功能需求

- **REQ-AI-001**: 当用户在答题过程中请求提示时，系统应调用AI模型生成与当前题目相关的知识点提示，不直接给出答案。
  - **验收标准**: 提示内容包含相关知识点说明和解题思路引导，不包含正确答案。

- **REQ-AI-002**: 系统应支持用户选择提示级别（轻度提示、中度提示、深度提示），不同级别提供不同详细程度的引导。
  - **验收标准**: 轻度提示仅给出知识点方向，中度提示增加解题思路，深度提示提供详细分析但仍不直接给出答案。

- **REQ-AI-003**: 系统应记录用户使用AI提示的次数和题目，用于学习分析中标识用户薄弱环节。
  - **验收标准**: 使用提示后记录完整保存，学习分析页面可查看提示使用统计。

- **REQ-AI-004**: 当AI模型服务不可用时，系统应优雅降级，显示"提示服务暂不可用"而不影响正常答题功能。
  - **验收标准**: AI服务异常时，提示按钮显示不可用状态，答题流程不受影响。

- **REQ-AI-005**: 系统应支持管理员配置AI模型的API密钥和服务端点。
  - **验收标准**: 管理员可在设置页面配置API密钥和端点，配置保存后AI提示功能可用。

- **REQ-AI-006**: 系统应限制每道题的AI提示请求次数（默认3次），防止过度依赖提示。
  - **验收标准**: 超过限制次数后提示按钮禁用，显示"本题提示次数已用完"。

#### 2.10.2 非功能需求

- **NFR-AI-001**: AI提示响应时间应不超过5秒。
- **NFR-AI-002**: AI提示功能不应影响答题页面的正常加载和交互。

### 2.11 移动端适配

#### 2.11.1 功能需求

- **REQ-MB-001**: 系统应采用响应式布局设计，在手机（宽度<768px）、平板（768px-1024px）和桌面（>1024px）设备上均提供良好的浏览和操作体验。
  - **验收标准**: 在iPhone、Android手机、iPad等设备上访问，页面布局自适应，内容无溢出或遮挡。

- **REQ-MB-002**: 系统应在移动端优化触控交互，包括按钮和选项的触控区域不小于44×44像素，支持滑动手势切换题目。
  - **验收标准**: 移动端所有可点击元素触控区域达标，左右滑动可切换上一题/下一题。

- **REQ-MB-003**: 系统应在移动端提供适配的答题界面，选项垂直排列，答题进度条和计时器在顶部固定显示。
  - **验收标准**: 移动端答题界面选项垂直排列，进度条和计时器始终可见。

- **REQ-MB-004**: 系统应在移动端优化统计图表展示，采用适配小屏幕的图表样式，支持触摸缩放和滑动查看。
  - **验收标准**: 移动端图表可正常显示，支持双指缩放和单指滑动。

- **REQ-MB-005**: 系统应支持移动端的离线缓存策略，已访问的题目和用户数据在弱网或断网环境下可查看。
  - **验收标准**: 断网后已访问的题目仍可查看，网络恢复后自动同步数据。

#### 2.11.2 非功能需求

- **NFR-MB-001**: 移动端首屏加载时间应不超过2秒（4G网络环境）。
- **NFR-MB-002**: 移动端页面Lighthouse性能评分应不低于70分。

### 2.12 部署与运维

#### 2.12.1 功能需求

- **REQ-DP-001**: 系统应支持Docker容器化部署，提供Dockerfile和docker-compose.yml配置文件。
  - **验收标准**: 执行docker-compose up命令后，系统完整启动并可正常访问。

- **REQ-DP-002**: 系统应使用Nginx作为反向代理服务器，Gunicorn作为应用服务器。
  - **验收标准**: Nginx正确代理请求至Gunicorn，静态文件由Nginx直接服务。

- **REQ-DP-003**: 系统应支持HTTPS访问，使用Let's Encrypt或自签名SSL证书。
  - **验收标准**: 通过https://访问系统，浏览器显示安全连接标识。

- **REQ-DP-004**: 系统应支持CI/CD自动部署，通过GitHub Actions实现代码推送后自动构建、测试和部署。
  - **验收标准**: 代码推送到主分支后，CI/CD流水线自动执行构建、测试和部署。

- **REQ-DP-005**: 系统应提供健康检查接口，用于监控服务运行状态。
  - **验收标准**: 访问健康检查接口返回200状态码和服务状态信息。

- **REQ-DP-006**: 系统应预留扩展接口，支持未来接入更多题型、更多AI模型和第三方认证等功能。
  - **验收标准**: 扩展接口定义清晰，新增题型无需修改核心答题逻辑，AI模型可插拔替换。

#### 2.12.2 非功能需求

- **NFR-DP-001**: 系统启动时间应不超过30秒（Docker容器启动）。
- **NFR-DP-002**: 系统应支持在2GB内存的Linux服务器上正常运行。

### 2.13 GitHub项目管理与代码更新记录

#### 2.13.1 功能需求

- **REQ-GH-001**: 系统项目代码应托管在GitHub仓库，仓库包含完整的项目源码、配置文件和文档。
  - **验收标准**: GitHub仓库可正常访问，包含前端、后端、部署配置等完整代码。

- **REQ-GH-002**: 系统应使用语义化版本号（Semantic Versioning）管理发布版本，版本号格式为MAJOR.MINOR.PATCH。
  - **验收标准**: 每次发布有明确的版本号标签（Git Tag），CHANGELOG.md记录版本变更内容。

- **REQ-GH-003**: 系统应维护CHANGELOG.md文件，记录每个版本的更新内容，包括新增功能、修复问题、破坏性变更。
  - **验收标准**: CHANGELOG.md按版本倒序排列，每个版本条目包含日期、类型和详细描述。

- **REQ-GH-004**: 系统应使用Git分支策略管理开发流程，主分支（main）为稳定版本，开发分支（develop）为集成分支，功能分支（feature/*）为开发分支。
  - **验收标准**: 分支结构清晰，功能开发在feature分支进行，合并至develop后测试，最终合并至main发布。

- **REQ-GH-005**: 系统应通过GitHub Actions实现CI/CD流水线，包括代码检查（lint）、单元测试、构建和自动部署阶段。
  - **验收标准**: 每次推送触发CI流水线，PR合并前必须通过所有检查，主分支更新触发CD部署。

- **REQ-GH-006**: 系统应提供README.md项目说明文档，包含项目简介、技术栈、安装部署指南、开发指南和API文档链接。
  - **验收标准**: 按照README.md指引可完成项目本地开发和部署。

- **REQ-GH-007**: 系统应使用.gitignore文件排除敏感信息（密钥、环境变量）、编译产物和依赖目录，确保代码仓库安全整洁。
  - **验收标准**: 仓库中不包含.env、__pycache__、node_modules等文件，无密钥泄露。

#### 2.13.2 非功能需求

- **NFR-GH-001**: CI流水线完整执行时间应不超过10分钟。
- **NFR-GH-002**: 代码提交前必须通过lint检查和单元测试（pre-commit hook或CI门禁）。
